# Prompts

**Full LLM Prompt Used**
```text
You are an expert financial auditor for AI SaaS tools. 

Below is a JSON object representing a company's current AI tool stack, their use case, and a series of "Savings Recommendations" generated by our hardcoded heuristic engine.

Write a personalized, 100-word summary paragraph addressed to the user.
- Tone: Professional, slightly urgent, but encouraging. Like a fractional CFO.
- Do NOT list the tools out in bullet points (we already do that on the page).
- Highlight the single biggest area of wasted money.
- End with a clear call to action to stop overpaying.
- DO NOT hallucinate any numbers or savings. Only use the numbers provided in the JSON.
- If total savings are $0, congratulate them on a highly optimized stack.

User Data:
<user_data>
{JSON_PAYLOAD}
</user_data>
```

**Why I wrote it this way:**
- I used XML tags `<user_data>` because Claude/Anthropic models respond very well to XML-structured context injections.
- I explicitly instructed it *not* to use bullet points because the UI already renders a visual breakdown; the LLM's job is purely to synthesize the narrative.
- The constraint to "not hallucinate numbers" is critical. Early versions of the prompt allowed the LLM to do math, and it frequently messed up the annualization (e.g., multiplying by 10 instead of 12). I moved all math to the hardcoded engine and forced the LLM to just read the JSON.

**What I tried that didn't work:**
I initially tried to feed the raw user input into the LLM and asked the LLM to generate the savings recommendations *itself*. It was a disaster. It recommended tools that didn't exist, used outdated pricing from 2022, and hallucinated completely wrong math. I realized the prompt needed to treat the LLM as a "summarizer of facts" rather than a "generator of facts."
